# Security Policy

## Supported Versions

Dbugr is early-stage open source software. Security fixes target the current `main` branch and the latest GitHub Release.

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive reports.

Email the maintainer at `security@dbugr.ai` with:

- a clear description of the issue
- reproduction steps or proof of concept, if available
- affected platform and version
- whether local files, screenshots, API keys, or account data may be exposed

If that mailbox is not yet active, use GitHub private vulnerability reporting for this repository.

## Security Notes

- Dbugr's desktop app is local-first. Claude, Codex, and Cursor credentials are intended to remain on the user's Mac.
- Screen captures and annotations may contain sensitive product or customer data. Do not attach private screenshots to public issues.
- The packaged macOS app may require Screen Recording permission before capture works.
